# Manual Testing Project – E-commerce Store

Celem projektu było przetestowanie podstawowych funkcjonalności sklepu internetowego.

## Zakres testów:
- logowanie użytkownika
- przeglądanie produktów
- dodawanie produktów do koszyka
- edycja koszyka
- składanie zamówienia
- wylogowanie

## Typy testów:
- testy funkcjonalne
- testy negatywne
- testy regresji

## Edge Cases:
- bardzo duża liczba produktów w koszyku
- szybkie wielokrotne kliknięcia
- brak internetu podczas składania zamówienia
- niepoprawne dane płatności
- timeout systemu

## Ryzyka:
- niepoprawny produkt w zamówieniu -> niezadowolony klient
- brak walidacji danych -> błędy systemowe
- duże obciążenie -> spadek wydajności
